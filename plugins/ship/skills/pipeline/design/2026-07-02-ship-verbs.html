/ship verbs — explicit process pins

Design spec · 2026-07-02 · ships as an edit to skills/ship/SKILL.md

TL;DR: /ship keeps sizing the ceremony itself — nothing changes for the bare form or the auto-trigger. But now a first-word verb pins the process explicitly: express pins the tweak lane, design pins the full gated design walkthrough, skill drops into skill authoring. Verbs are overrides, not a menu. Every one of them starts the same way: a worktree off main via worktrunk.

Dispatch — one decision, up front

/ship …→ first word a verb?→ yes: process pinned, sizing skipped / no: auto-size EXPRESS · SELF-DIRECTED · GATED (today's behavior)

The three verbs

/ship express <tweak>

Pins EXPRESS: worktree → change → repo gates → self-drive → merge → dev. No spec, no plan, no stops. Pins ceremony down, never safety down — schema, auth, money, or a taste call mid-flight still promotes, verb or not.

/ship design <idea>

Pins GATED with the full design walkthrough front-loaded: questions one at a time, 2–3 approaches with a recommendation, HTML spec + live mockup, GATE 1 — then the normal pipeline through to dev. The verb asserts taste is in play; it is never downgraded to SELF-DIRECTED.

/ship skill <what>

Authors or evolves a skill instead of product code. Reuses the installed skill-creator discipline (pressure-test scenario first). Repo-homed skills ride the normal rails; ship-plugin skills also get the plugin-cache deploy so running sessions pick them up.

Where a skill lives decides its ceremony

Skill homeRailsVerification

a project's .claude/skills/that repo's wt worktree → merge
personal ~/.claude/skills/no repo ceremony — write + test
The worktrunk rule is universal. Every verb, like every lane, begins at stage 0: wt switch --create feature/<slug> --no-cd --format=json -y off main. No verb ever edits a primary checkout.
What deliberately didn't make the cut
  • /ship status and /ship promote — proposed, rejected (Pete, 2026-07-02): core three only, keep the menu minimal. Promotion stays a separate human ritual outside ship.
  • /ship fix, /ship build, /ship review — the lanes already cover these; a fat verb menu is how CLIs rot.
  • Mandatory verbs — rejected: bare /ship <idea> and the auto-trigger keep sizing themselves.
  • A separate skill file per verb — rejected: one ~40-line section in the ship SKILL.md; the verbs share every rail with the lanes.
