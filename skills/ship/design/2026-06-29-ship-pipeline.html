An idea enters on the left; a reviewed feature lands on main on the right. You touch three points; the machine does the rest.

The whole thing in one breath

DISCOVER (your taste, up front) → the machine plans, checks in once, builds, verifies, and merges. You're heavy in stage 1, glance at one card in the middle, and come back at the end. Everything you see is HTML; you only ever read the meta, never the full spec.

Underneath, every feature runs in its own worktrunk worktree — isolated, auto-provisioned, merged and cleaned up in one step. You never touch git. main only ever moves in clean, reviewed chunks.

Tiny & watched → merge. Substantial → PR + card + wait.

Your status line, reimagined

It stops being branch-centric and starts telling you which feature, what stage, which tree. The stage is written into the worktree's state as /ship advances; the line just reads it (fast, local — no lag).

# idle on main 📁 homezero · Opus 4.8 · ⚡ high 🌳 main · clean # mid-build 📁 homezero · Opus 4.8 · ⚡ high 🚢 saved-filters · 🔨 build 4/5 · +42 −8 # waiting on your design call 📁 homezero · Opus 4.8 · ⚡ high 🚢 saved-filters · 🔍 discover · 💬 your call # PR up for review 📁 homezero · Opus 4.8 · ⚡ high 🚢 saved-filters · 👀 review · PR #128

Dropped: version, "20m ago", the bare branch line, "2 uncommitted" (→ real diff count). Stage glyphs: 🔍 discover → 📐 plan → ✅ go → 🔨 build → 👀 review. The all-trees view stays wt list + FleetView.

Where each skill fires

brainstormingDiscover — pins intent, PM-framed, one question at a time.
impeccableDiscover — HTML design sprint (pix imagery). Skipped for non-visual features. init already done for homezero.
writing-plansPlan — the implementation plan (HTML artifact).
ponytail ×2Plan (scope critic, primary) + Build (posture). The cut-list rides onto the go-card.
routerBuild only — Opus driver splits each build task ~50/50 Opus ↔ GPT-5.5 (codex). Two engines, no Sonnet/Gemini. Self-tunes via a ledger. Never fires outside Build.
verify-before-doneBuild→Review — the agent runs the thing and watches it work before claiming done. The trust guard.
systematic-debuggingBuild — the failure fallback when a test goes red.
/code-review + ponytail-reviewReview — correctness + over-build, one automated pass.
worktrunk (wt)Stage 0 + merge — isolation, provisioning, one-step merge/cleanup. Invisible.

Deliberately left out

executing-plans (checkpoint-heavy — the opposite of hands-off), writing-skills (only used once, to build /ship itself), finishing-a-development-branch & using-git-worktrees (wt replaces both), strict TDD (tests as a build deliverable + the pre-merge npm test gate is enough; reserve test-first for money/auth), dispatching-parallel-agents (a power-up for big features, not a default).

How you're treated, and what you decide

Your lane

Product direction · design & taste · scope priorities · what the buyer experiences. Surfaced richly. Hard stops live here. You're a technical PM, not an engineer — technical decisions come translated, with a rec, for your call.

The machine's lane

Library choices, code structure, mechanics — handled, summarized in one plain line. Built on the standing stack (flue · Cloudflare · Convex · Clerk · Stripe · Next 16) without asking. Escalates only a choice that's both architectural and outside the canon.

Always-ship (no slash command required)

You won't have to type /ship. The skill auto-triggers on feature-shaped requests, and homezero's AGENTS.md makes it the default: "feature and multi-step change work runs through the ship flow — worktree, discover → plan → build → review; never edit main directly." A typo fix or a question still just gets done — no ceremony for the trivial.

v2 — the dockmaster loop

Once the single-feature engine is solid, wrap it in a self-paced supervisor loop — one long-lived /loop session in your FleetView fleet, running on your always-on Mac (it must be local — that's where the worktrees live). Each wake it advances every ship that can move without you, parks the ones at gates, and paces itself: tighten when busy, drift toward the 1h ceiling when idle, wake instantly on your replies.

The loop changes the clock, not the control. Your gates still gate — it advances work to them and stacks decisions for you to handle in batches. The tiny/safe class you already trust → it ships end-to-end; substantial → it parks for you. You stop being the thing that pushes the pipeline forward between gates.
